# OCAR WebShop Observe 训练总结

本次训练已正常完成，共运行 **150/150 steps**，总耗时约 **9 小时**，平均 **198.8 s/step**。训练过程中无报错，GPU 显存占用约 **120.7 GB reserved**，纯训练吞吐约 **1147 tok/s**，纯训练耗时约 **6.3 小时**。

## 一、总体结论

整体来看，本次训练过程稳定，`observe` 信号全程正常接入，相关指标持续记录且未干扰 GRPO 主训练流程。模型从初始阶段“完全不会做”逐步学会合法动作与基础交互，随后进入成功率加速提升阶段，并在中后期达到较高水平。验证集表现也呈现出清晰的阶段性上升趋势，说明策略学习与隐式 world model 改善是同步发生的。

从结果上看：

* **训练成功率（Episode SR）** 最终达到 **75.0%**
* **验证成功率（Val SR）** 峰值达到 **62.5%**
* 最优验证 checkpoint 出现在 **step 110**
* `wm_A < wm_B` 全程成立，说明带 observation context 的状态预测始终优于不带 context 的预测
* `obs_s_theta_mean` 总体下降，表明隐式 world model 对环境动态的建模能力在持续增强
* `delta_s_mean` 从接近 0 逐渐增大到约 `-0.17~-0.21`，完整呈现出 actor 从冷启动到系统性偏离 reference policy 的典型过程

## 二、关键训练节点

### 1. 初始与冷启动阶段

在初始 validation（step 0）时，模型 **Val SR = 0.0%**，说明一开始完全不具备完成任务的能力。
到 step 1 时，`delta_s_mean = -0.003`，偏离几乎为零，属于典型冷启动状态；同时 `valid_act` 仅 **45.5%**，说明连合法动作都还未完全学会。

### 2. 早期起步阶段

step 2 时，Episode SR 首次达到 **3.1%**，且 `valid_act` 快速跃升到 **98%**，说明模型首先学会了基本交互规则。
step 4 时，验证集首次非零，**Val SR = 3.1%**，标志着模型已经初步具备泛化到验证任务的能力。

### 3. 中期加速阶段

从 step 27 开始，Episode SR 首次突破 **5%**，达到 **9.4%**；同时 `wm_A` 与 `wm_B` 都开始下降，表示模型对环境状态转移的预测变得更准确。
到 step 42，`obs_s_theta_mean` 显著下降至 **2.153**，说明 observation-based NLL 已有明显改善。
step 72 时，Episode SR 达到 **34.4%**，进入明显的性能起飞阶段。

### 4. 高速增长与高位稳定阶段

step 87 时，Episode SR 提升到 **62.5%**，`obs_s_theta_mean` 创下新低 **2.009**，同时 episode length 缩短到 **8.6**，说明模型不仅更容易成功，而且完成任务更高效。
step 102 时，`wm_A` 进一步下降到 **1.833**，达到全程最低点，反映出 world model 预测能力达到较强水平。
step 117 与 132 时，Episode SR 分别达到 **71.9%** 与 **78.1%**，说明策略已进入高水平区间。

### 5. 后期波动阶段

在 step 146，训练末期仍保持较强表现，Episode SR 为 **65.6%**，Val SR 为 **56.2%**。
最终 step 149，Episode SR 为 **75.0%**，训练成功完成。
尽管后期存在一定波动，但总体仍维持在较高水平，说明模型已基本学成，只是受小 batch 和任务采样差异影响，单点指标会有抖动。

## 五、核心现象解释

### 1. Observe 信号工作正常

整个训练过程中，所有 `observe/*` 指标都持续记录，没有影响 GRPO 主干训练。这说明 observe 分支已经被正确接入，并且可以作为后续分析 world model 变化的重要观测窗口。

### 2. obs NLL 稳定下降

`obs_s_theta_mean` 从 **2.424** 下降到约 **2.15–2.20**，整体趋势清晰。虽然中间有局部波动，但总方向是下降的，说明模型对环境状态变化的预测能力在持续改善，这与成功率提升趋势一致。

### 3. delta_s 的冷启动到饱和过程完整出现

`delta_s_mean` 从 **-0.003** 逐步增长到 **-0.17~-0.21**。
这说明训练初期 actor 几乎与 ref 相同，随后逐渐形成系统性偏移，并在中后期进入相对稳定的偏离区间。这是一个很典型、也很健康的学习轨迹。

### 4. wm_A 始终小于 wm_B

`wm_A` 表示带 observation context 的预测误差，`wm_B` 表示不带 context 的预测误差。全程 `wm_A < wm_B`，而且差值长期稳定存在，说明 observation history 确实提供了额外有效信息，模型也成功利用了这些信息。

### 5. wm surprise 整体下降

`wm_A` 从约 **3.0** 下降到 **2.0–2.3**，说明模型对环境的“惊讶度”在减弱，即它对环境动态逐渐熟悉。这一趋势和 obs NLL 下降、SR 提升是相互一致的。

### 6. entropy 与 KL 的变化合理

entropy 并没有一路下降到塌缩，而是在中期重新升高到 **0.75–0.85** 的区间，说明策略仍保有足够探索性。
KL 从 **0.01** 增长到 **0.3–0.47**，结合 `kl_loss_coef=0.01` 的设定，这个增长速度是合理的，说明模型在稳步偏离 ref，而没有出现完全失控。

### 7. episode length 明显缩短

`ep_len` 从 **14.4** 缩短到 **7–8** 步，表明模型后期不仅能完成任务，而且效率更高，决策更直接。

## 六、后期波动的原因判断

在 step 111–130 区间，Episode SR 回落到 **39.6%**，虽然之后又恢复并进一步提升，但说明后期方差仍然不小。这个现象大概率不是训练崩坏，而更可能由以下因素造成：

* **batch=4 较小**，单次更新的统计波动较大
* WebShop 任务本身难度分布不均，不同 batch 抽到的任务差别可能很大
* 在高性能区间，策略已接近局部饱和，小扰动也更容易反映到 SR 波动上

因此，这种后期起伏更像是高方差下的正常震荡，而不是训练失败信号。
